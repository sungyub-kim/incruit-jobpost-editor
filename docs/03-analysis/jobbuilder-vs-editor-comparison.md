# 인크루트 채용공고 등록(JobBuilder) vs 에디터 비교 분석

> **Analysis Type**: Feature Comparison & Improvement Proposals
>
> **Project**: Incruit Jobpost Editor
> **Date**: 2026-02-07
> **비교 대상**: recruiter.incruit.com/jobpost/jobbuilder.asp (공고등록 8단계 위자드)
> **분석 대상**: localhost:8080 (Incruit Jobpost Editor v1.0)

---

## 1. 개요

인크루트 공식 채용공고 등록 시스템(JobBuilder)은 8단계 위자드 방식으로 구조화된 데이터를 입력받아 채용공고를 생성한다. 반면 우리 에디터는 원문 텍스트를 AI로 변환하여 HTML 템플릿 + KV 이미지를 생성하는 도구이다.

두 시스템은 **역할이 다르지만 겹치는 영역**(템플릿 적용, KV 생성)이 있으며, JobBuilder의 UX 패턴에서 우리 에디터에 적용할 수 있는 개선점이 다수 발견되었다.

---

## 2. JobBuilder 8단계 구조 분석

| 단계 | 이름 | 주요 입력 필드 | 에디터 대응 |
|------|------|---------------|------------|
| 1 | **모집내용** | 공고제목, 모집부문, 직종/직무, 고용형태, 모집인원, 직급/직책 | 부분 대응 (AI가 추출) |
| 2 | **상세업무** | 담당업무(텍스트), 직무기술(태그), 근무부서 | 원문에서 AI 추출 |
| 3 | **지원자격** | 학력, 경력(신입/경력/년수), 필수/우대조건 | 원문에서 AI 추출 |
| 4 | **근무조건** | 근무지역, 지하철역, 급여조건, 복리후생 | 원문에서 AI 추출 |
| 5 | **채용방식 및 절차** | 접수기간, 접수방법, 접수양식, 전형단계 | 원문에서 AI 추출 |
| 6 | **채용담당자정보** | 담당자명, 연락처, 이메일 | 미대응 |
| 7 | **템플릿 적용** | 디자인 선택, WYSIWYG 에디터, 문구 추가(태그/회사소개) | **핵심 경쟁 영역** |
| 8 | **지원율 높이기** | 상품선택(유료 광고 패키지) | 해당 없음 |

## 3. 기능별 상세 비교

### 3.1 템플릿 시스템 (7단계 vs 에디터)

| 항목 | JobBuilder | 에디터 | 비교 |
|------|-----------|--------|------|
| 디자인 템플릿 수 | 9+ (썸네일 선택) | 6 (드롭다운) | JobBuilder 우위 |
| 선택 방식 | **비주얼 썸네일** 그리드 | 텍스트 드롭다운 | JobBuilder 직관적 |
| WYSIWYG 에디터 | 내장 (글꼴, 크기, 정렬, 색상, URL, 표) | 플로팅 툴바 (B/I/U/H1-H3/링크) | JobBuilder 풍부 |
| Editor/HTML/TEXT 전환 | 3탭 전환 | Split View / 원문 / 미리보기 / HTML 4탭 | **에디터 우위** |
| 문구 추가 | 태그 ON/OFF, 회사소개 ON/OFF | 없음 | JobBuilder 우위 |
| 모집부문 행 추가 | 전용 UI 버튼 | 없음 (AI 자동) | 용도 차이 |
| 실시간 미리보기 | 동일 영역 내 | 별도 패널 (더 넓음) | **에디터 우위** |

### 3.2 키비주얼 (KV) 비교

| 항목 | JobBuilder | 에디터 | 비교 |
|------|-----------|--------|------|
| KV 이미지 | 고정 디자인 (템플릿에 포함) | **완전 커스텀** (프리셋 5종 + 세부 설정) | **에디터 압도적 우위** |
| 텍스트 편집 | 제한적 (템플릿 고정) | 4필드 개별 설정 (폰트/색상/위치/굵기) | **에디터 우위** |
| 배경 이미지 | 템플릿 고정 | 업로드/URL/검색/업종자동선택 | **에디터 우위** |
| 로고 | 미지원 | 로고 업로드 + 위치 설정 | **에디터 우위** |
| 내보내기 | 없음 (서버 렌더링) | PNG 2x/4x, JPG, HTML | **에디터 우위** |
| 레이아웃 | 1종 고정 | 오버레이/분할 + 효과(그라데이션/대각선/곡선) | **에디터 우위** |

### 3.3 AI 기능 비교

| 항목 | JobBuilder | 에디터 | 비교 |
|------|-----------|--------|------|
| AI 초안 생성 | "+ 새로운 초안 생성" 버튼 (담당업무) | AI 변환 (전체 공고 변환) | 용도 차이 |
| AI 범위 | 담당업무 텍스트 1개 필드 | 전체 채용공고 HTML 변환 | **에디터 우위** |
| AI 모델 | 비공개 (인크루트 내부) | Claude/GPT-4o/Gemini 선택 가능 | **에디터 우위** |
| 원문 보존 | 해당 없음 | 원문 100% 보존 검증 시스템 | **에디터 우위** |

### 3.4 입력/추출 비교

| 항목 | JobBuilder | 에디터 | 비교 |
|------|-----------|--------|------|
| 입력 방식 | 구조화된 폼 (필드별) | 비구조화 (원문 통째로) | 용도 차이 |
| URL 추출 | "이전 공고 불러오기" | 멀티 플랫폼 URL 추출 (11개 사이트) | **에디터 우위** |
| 파일 업로드 | 미지원 | DOCX/PDF/XLSX/HWP 등 8종 | **에디터 우위** |
| 구조 데이터 | 직종/직무 태그, 고용형태 체크박스 | 없음 (비구조) | JobBuilder 우위 |

## 4. 제안사항 (에디터 개선점)

### 4.1 즉시 적용 가능 (Quick Win)

#### P1. 템플릿 비주얼 선택기
- **현재**: 텍스트 드롭다운 (`Standard (기본형)`)
- **개선**: JobBuilder처럼 **썸네일 그리드** 방식으로 변경
- **구현**: 각 템플릿의 미리보기 이미지를 생성하여 카드 형태로 표시
- **효과**: 사용자가 결과물을 미리 예측하고 선택할 수 있음
```
현재:  [Standard (기본형) ▼]
개선:  [썸네일1] [썸네일2] [썸네일3]
       Standard  Modern  Corporate
       [썸네일4] [썸네일5] [썸네일6]
       Incruit   Creative Compact
```

#### P2. 구조화 데이터 사이드 패널
- **현재**: AI가 원문에서 자동 추출하지만, 추출 결과를 사용자가 확인/수정할 수 없음
- **개선**: AI 변환 후 구조화 데이터를 사이드 패널에 표시
- **표시 항목**: 기업명, 직종, 고용형태, 경력 조건, 근무지역, 마감일
- **효과**: 사용자가 AI 추출 결과를 검증하고, KV 자동 채우기 정확도 향상
```
┌─ 추출 정보 ──────────┐
│ 기업명: (주)인크루트   │
│ 직종: 서비스개발       │
│ 고용형태: 정규직       │
│ 경력: 7~9년           │
│ 근무지역: 서울         │
│ 마감일: 2026-03-09    │
└───────────────────────┘
```

#### P3. 문구 추가 토글 (태그/회사소개)
- **현재**: 없음
- **개선**: JobBuilder의 "태그 ON/OFF", "회사소개 ON/OFF" 패턴 채용
- **구현**: 템플릿 하단에 선택적 섹션 추가
  - 태그: 직무기술 키워드를 배지 형태로 표시
  - 회사소개: 기업 소개 문구 블록 ON/OFF
- **효과**: 고객사가 추가 정보를 쉽게 토글

### 4.2 중기 개선 (Medium-Term)

#### P4. 전형단계 비주얼 에디터
- **현재**: AI가 텍스트로 추출 (`서류전형 > 면접진행 > 최종심사 > 최종합격`)
- **개선**: JobBuilder처럼 단계별 시각적 흐름도(stepper) 위젯
- **구현**: 드래그로 단계 추가/삭제/순서변경 가능
- **효과**: 전형 프로세스가 시각적으로 명확

#### P5. 복리후생 구조화 입력
- **현재**: 원문 텍스트 그대로
- **개선**: 카테고리별 분류 표시 (기본사항, 근무환경/편의, 교육/여가, 보장/보상)
- **참고**: JobBuilder 4단계의 카테고리 분류 패턴

#### P6. WYSIWYG 에디터 강화
- **현재**: 플로팅 툴바 (B/I/U/H1-H3/링크/서식제거)
- **개선 항목**:
  - 글꼴 선택 (JobBuilder는 글꼴/크기 드롭다운 제공)
  - 텍스트 색상 변경
  - 표 삽입/편집
  - 정렬 (좌/중/우)
  - 이미지 삽입
- **우선순위**: 표 편집 > 정렬 > 색상 > 글꼴 > 이미지

#### P7. "이전 공고 불러오기" 기능
- **현재**: 매번 새로 입력
- **개선**: 최근 작업한 공고 목록을 localStorage에 저장하여 재사용
- **구현**: "최근 작업" 드롭다운 또는 히스토리 패널
- **효과**: 반복 작업 시간 절약

### 4.3 장기 개선 (Long-Term)

#### P8. 인크루트 API 직접 연동
- **현재**: HTML 복사 → JobBuilder 7단계에 수동 붙여넣기
- **개선**: 에디터에서 직접 인크루트 채용공고 등록 API 호출
- **효과**: 복사/붙여넣기 단계 제거, 원클릭 등록
- **고려사항**: recruiter.incruit.com 인증 토큰 연동 필요

#### P9. 구조화 데이터 → JobBuilder 매핑
- **현재**: 에디터는 비구조화 HTML만 생성
- **개선**: AI 추출 결과를 JobBuilder 1~6단계 필드에 맞는 JSON으로 출력
- **활용**: 향후 API 연동 시 구조화 데이터로 자동 채움 가능
- **출력 예시**:
```json
{
  "step1_모집내용": {
    "공고제목": "서비스개발 경력 채용",
    "모집부문": "서비스개발",
    "직종직무": ["물류관리", "네트워크구축운영관리"],
    "고용형태": "정규직",
    "모집인원": "0명"
  },
  "step3_지원자격": {
    "학력": "학력무관",
    "경력": "경력 7~9년"
  },
  "step5_채용방식": {
    "접수마감일": "2026-03-09",
    "전형단계": ["서류검토", "면접진행", "최종심사", "최종합격"]
  }
}
```

#### P10. 멀티 모집부문 지원
- **현재**: 공고 1건 = 1개 결과물
- **개선**: JobBuilder처럼 1개 공고에 여러 모집부문 입력 → 각각 다른 담당업무/자격요건
- **구현**: 모집부문 탭 + 부문별 콘텐츠 영역

## 5. 에디터 고유 강점 (유지/강화)

JobBuilder 대비 에디터만의 차별점으로, 반드시 유지해야 할 강점:

| # | 강점 | 설명 |
|---|------|------|
| 1 | **멀티 플랫폼 URL 추출** | 잡코리아, 사람인, 원티드 등 11개 사이트에서 추출 |
| 2 | **파일 다이렉트 변환** | DOCX/PDF/HWP 등 원문 파일을 바로 변환 |
| 3 | **AI 모델 선택** | Claude/GPT/Gemini 중 선택 가능 |
| 4 | **KV 완전 커스텀** | 5 프리셋 + 세부 설정 (위치/색상/폰트/효과) |
| 5 | **원문 보존 검증** | AI 변환 후 원문 누락 자동 감지 |
| 6 | **다크 모드** | JobBuilder에 없는 다크 테마 |
| 7 | **Split View** | 원문/미리보기 동시 비교 |
| 8 | **오프라인 사용** | 로컬 서버만으로 동작 (인터넷 불필요 - AI 제외) |

## 7. 결론

**에디터의 핵심 가치**는 "비구조화 원문 → 구조화 HTML + KV"로의 **AI 변환**에 있으며, 이는 JobBuilder가 제공하지 않는 고유한 기능이다.

반면 JobBuilder에서 배울 수 있는 것은 **구조화된 UX 패턴**(비주얼 선택, 토글 옵션, 단계별 흐름도)이며, 이를 에디터에 점진적으로 도입하면 사용성이 크게 향상될 것이다.

특히 **P1(템플릿 비주얼 선택기)**과 **P2(구조화 데이터 패널)**은 최소 노력으로 최대 효과를 낼 수 있는 개선 항목이다.
