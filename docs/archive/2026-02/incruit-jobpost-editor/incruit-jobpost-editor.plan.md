# Plan: incruit-jobpost-editor

> 채용공고 문서 → 인크루트 템플릿 변환 에디터

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Incruit Jobpost Editor** - 채용공고 템플릿 에디터

### 1.2 목적
다양한 형식의 채용공고 문서(HWP, Word, Excel, PDF, HTML, TEXT)를 인크루트 템플릿으로 변환하고, 디자이너가 쉽게 커스터마이징할 수 있는 웹 기반 에디터

### 1.3 대상 사용자
- 사내 디자이너 (윈도우/맥 혼용)
- Chrome 브라우저 사용자

### 1.4 핵심 원칙
1. **원문 보존**: 문서 형식과 텍스트를 정확히 유지 (오타/환각 금지)
2. **디자인 커스터마이징**: 키비주얼, 불릿 스타일, 디자인 팔레트 수정 가능
3. **템플릿 기반**: 인크루트 표준 템플릿에 맞춰 출력

## 2. 핵심 요구사항

### 2.1 레이아웃 구조 (업데이트)
```
┌─────────────┬──────────────────────────────────┬─────────────────┐
│   1단       │             2단                  │      3단        │
│ Template    │     Editor (2분할)               │   AI Chat       │
│ & Settings  │                                  │                 │
│             │ ┌──────────────┬───────────────┐ │                 │
│ - 채용공고번호│ │   원문 입력   │  템플릿 출력  │ │ - AI Agent      │
│ - 템플릿 선택 │ │   (복붙)     │   (Preview)   │ │ - 대화 기록      │
│ - 키비주얼   │ │              │               │ │ - 변환 명령      │
│ - 컬러팔레트 │ └──────────────┴───────────────┘ │                 │
│ - 불릿 스타일 │                                  │                 │
└─────────────┴──────────────────────────────────┴─────────────────┘
```

### 2.2 기능 요구사항

#### 1단: 설정 패널
| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 채용공고 번호 입력 | 필수 | 인크루트 공고번호 (ex: 12345678) |
| 템플릿 선택 | 필수 | 여러 인크루트 템플릿 중 선택 |
| 키비주얼 설정 | 필수 | 이미지 URL 또는 업로드 |
| 컬러 팔레트 | 필수 | Primary, Secondary, Accent 색상 |
| 불릿 스타일 | 필수 | 체크, 화살표, 숫자 등 |
| 폰트 설정 | 선택 | 제목/본문 폰트 선택 |

#### 2단: 에디터 (2분할)
| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 원문 입력 (좌) | 필수 | 복사-붙여넣기로 원문 입력 |
| 템플릿 출력 (우) | 필수 | 선택한 템플릿으로 변환된 결과 |
| 원문 형식 감지 | 필수 | 제목/본문/리스트 자동 감지 |
| 실시간 미리보기 | 필수 | 변환 결과 즉시 반영 |
| HTML 코드 보기 | 선택 | 최종 HTML 코드 확인/복사 |

#### 3단: AI 채팅
| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 문서 분석 | 필수 | AI가 원문 구조 분석 |
| 템플릿 변환 | 필수 | AI가 템플릿에 맞게 변환 |
| 디자인 제안 | 선택 | 컬러, 레이아웃 제안 |
| 오류 검증 | 필수 | 원문과 출력물 비교 검증 |

### 2.3 문서 입력 방식
```
✅ 복사-붙여넣기 방식 채택

장점:
- 서버 불필요
- 모든 문서 형식 지원 (HWP, Word, Excel, PDF 등)
- 사용자가 원하는 부분만 선택 가능
- 형식 보존 (표, 리스트 등)

워크플로우:
1. 원본 문서 열기 (HWP, Word 등)
2. 내용 전체 선택 (Ctrl+A) 후 복사 (Ctrl+C)
3. 에디터 원문 입력창에 붙여넣기 (Ctrl+V)
4. AI가 구조 분석 후 템플릿 적용
```

## 3. 템플릿 시스템

### 3.1 인크루트 템플릿 구조
```html
<!-- 기본 템플릿 구조 -->
<div class="incruit-jobpost" data-template="standard">
  <!-- 키비주얼 -->
  <header class="key-visual">
    <img src="{keyVisualUrl}" alt="채용공고 이미지">
  </header>

  <!-- 회사 정보 -->
  <section class="company-info">
    <h1>{companyName}</h1>
    <p class="tagline">{tagline}</p>
  </section>

  <!-- 모집 내용 -->
  <section class="job-details">
    <h2>모집 부문</h2>
    {jobContent}
  </section>

  <!-- 자격 요건 -->
  <section class="requirements">
    <h2>자격 요건</h2>
    {requirements}
  </section>

  <!-- 혜택 -->
  <section class="benefits">
    <h2>복리후생</h2>
    {benefits}
  </section>

  <!-- 지원 방법 -->
  <section class="apply-info">
    <h2>지원 방법</h2>
    {applyInfo}
  </section>
</div>
```

### 3.2 디자인 변수
```css
:root {
  /* 컬러 팔레트 */
  --color-primary: #0066cc;
  --color-secondary: #333333;
  --color-accent: #ff6600;
  --color-background: #ffffff;
  --color-text: #333333;

  /* 폰트 */
  --font-heading: 'Pretendard', sans-serif;
  --font-body: 'Pretendard', sans-serif;

  /* 불릿 스타일 */
  --bullet-style: '✓'; /* ✓, ●, ▶, 숫자 */
}
```

### 3.3 템플릿 종류 (예시)
| 템플릿 | 설명 | 특징 |
|--------|------|------|
| Standard | 기본형 | 심플한 레이아웃 |
| Modern | 모던형 | 큰 키비주얼, 여백 많음 |
| Corporate | 기업형 | 정돈된 표 형식 |
| Creative | 크리에이티브 | 자유로운 레이아웃 |
| Compact | 간결형 | 정보 밀도 높음 |

## 4. AI 변환 프로세스

### 4.1 원문 분석
```javascript
// AI가 분석하는 요소
{
  sections: [
    { type: 'title', content: '○○기업 신입/경력 채용' },
    { type: 'subtitle', content: '함께 성장할 인재를 찾습니다' },
    { type: 'heading', content: '모집 부문' },
    { type: 'list', items: ['개발팀', '마케팅팀', '디자인팀'] },
    { type: 'table', data: [...] },
    { type: 'paragraph', content: '...' }
  ],
  metadata: {
    companyName: '○○기업',
    hasTable: true,
    hasList: true,
    estimatedSections: 5
  }
}
```

### 4.2 변환 규칙
```
원문 요소 → 템플릿 매핑

- 첫 번째 큰 제목 → 회사명/공고 제목
- "모집", "채용" 포함 제목 → 모집 부문 섹션
- "자격", "요건", "우대" 포함 → 자격 요건 섹션
- "복리", "혜택", "급여" 포함 → 복리후생 섹션
- "지원", "접수", "방법" 포함 → 지원 방법 섹션
- 리스트 → 불릿 리스트 (스타일 적용)
- 표 → 테이블 또는 카드 형식
```

### 4.3 검증 프로세스
```
1. 텍스트 비교: 원문의 모든 텍스트가 출력에 존재하는지
2. 구조 검증: 섹션 누락 없는지
3. 오탈자 검증: 원문과 정확히 일치하는지
4. 형식 보존: 표, 리스트 등 구조 유지
```

## 5. 기술 스택

### 5.1 프론트엔드
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | Vanilla JS | 서버 없이 정적 배포 |
| 에디터 | CodeMirror 6 / contenteditable | 복붙 지원 |
| 스타일링 | Tailwind CSS | 빠른 UI 개발 |
| AI | OpenAI/Claude API | 문서 분석 및 변환 |

### 5.2 서버
```
❌ 서버 불필요 - 순수 클라이언트 앱

복사-붙여넣기 방식으로 문서 변환 처리
AI API는 클라이언트에서 직접 호출 (사용자 API 키)
```
